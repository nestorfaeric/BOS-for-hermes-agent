# Skill: Chase

Augmenter le revenue quand le goulot est **acquisition / monétisation** — pas le fondateur (Mindset), pas les opérations / qualité (Digestion). Déclenché quand `bos-diagnosis` identifie un goulot de type Chase en phase Scale.

## Objectif

1. **Cartographier** les leviers (acquisition, conversion, monétisation, capacité) avec effort / impact.
2. **Ancrer** le **palier** de CA et le **playbook** correspondant.
3. **Produire** une **roadmap 90 jours** (3 x 30j : Fondations, Accélération, Stabilisation) + budget + risques.
4. **Lister** **2-3 quick wins** cette semaine.
5. **Traiter** la diversification (2e business) **uniquement** si le sujet est posé et les prérequis sont remplis.
6. **Mettre à jour** les fichiers Core et sauver le plan dans `workspace/BOS/Output/`.

**Critère de succès :** une direction claire, **un levier dominant** par phase, et des actions vérifiables sur 7 jours.

## Croyances

- **Un levier à la fois.** Pas trafic + conversion + produit en parallèle — stabiliser, puis suivant.
- **Les paliers de CA ont des règles différentes.** Ce qui marche à 10K casse souvent à 100K — diagnostiquer le palier avant les tactiques.
- **Le pricing est souvent le levier le plus sous-utilisé.** Hausse de prix, packaging, upsell — souvent plus rapide que « plus de trafic ».
- **Chasse vs digestion.** Si la livraison ou la satisfaction plombent, **ne pas** compenser par plus d'acquisition — renvoyer vers `bos-digestion` et signaler le risque dans le plan.
- **Croissance ne veut pas dire scaling** si ça multiplie les heures fondateur sans levier — le dire explicitement.

## Process

### Phase 1 — Analyse des leviers de croissance

Pour **chaque levier** pertinent, produire une ligne :

- **État actuel** (chiffres ou qualitatif précis)
- **Opportunité** (ce qui est sous-exploité)
- **Effort** — L / M / H
- **Impact** — L / M / H
- **Recommandation** — faire maintenant / plus tard / ne pas faire

**Catégories :**

**A. Acquisition** — optimiser le canal existant ; 2e canal ; referral / affiliation ; partenariats ; pub ; contenu / SEO.

**B. Conversion** — process / copy vente ; preuves sociales ; friction achat ; pricing / packaging ; garantie.

**C. Monétisation** — prix ; upsell / cross-sell ; offre premium ; rétention / churn ; fréquence d'achat.

**D. Capacité** — recruter / déléguer ; automatiser ; productiser ; SOPs ; livraison groupe (souvent pont vers `bos-digestion` — le nommer si c'est le vrai goulot).

**Règle de priorisation :** un **levier dominant** pour la phase en cours ; les autres en backlog daté.

### Phase 2 — Playbook par palier

Utiliser le **CA mensuel récurrent** (ou équivalent) pour ancrer. Adapter au contexte — ce sont des **patterns**, pas une religion.

| Palier | Focus typique |
|--------|---------------|
| **3K - 10K** | Maîtriser **UN** canal (~80% de l'effort dessus) ; ~10 leads/semaine stables ; premières SOPs ; première embauche ciblée. |
| **10K - 30K** | Livraison **sans** fondateur sur une partie du flux ; 1-2 embauches ops ; **2e canal** seulement quand le 1er est stable ; upsell ; **augmenter les prix** si sous-positionné. |
| **30K - 100K** | Fondateur **manager**, pas doer sur le coeur ; équipe structurée ; **KPI dashboard** ; productiser ; partenariats. |
| **100K+** | Plusieurs produits / segments ; équipe de direction ; marque ; acquisitions / partenariats stratégiques. |

**Paliers de référence :** Traction 0-3K ; Foundation 3-10K ; Growth 10-30K ; Scale 30-100K ; Expansion 100K+.

### Phase 3 — Roadmap 90 jours

Structurer en **3 phases de 30 jours** :

1. **Fondations** — Ce qui doit être vrai avant d'accélérer (tracking, offre/prix, canal #1, SOPs minimales).
2. **Accélération** — Pousser le levier choisi ; métriques hebdo ; budget acquisition si applicable.
3. **Stabilisation** — Marges, rétention, documentation, éviter le tout-casser.

Pour **chaque** phase de 30 jours :

- **Focus** (une phrase)
- **Actions** (liste vérifiable)
- **Milestone** (critère de succès)
- **KPIs** (3 max)

Ajouter :

- **Budget** — acquisition, outils, équipe (même approximatif)
- **Risques** — trésorerie, dépendance canal, saturation livraison, baisse qualité

### Phase 4 — Quick wins (cette semaine)

Proposer **2-3** actions à **impact rapide**, par exemple (adapter au contexte) :

- Hausse de prix ou test A/B sur l'offre principale
- Programme **referrals** ou relance **anciens clients**
- Upsell au checkout ou bump
- Refonte headline / above-the-fold landing
- Relance pipeline existant avant nouveau trafic

### Phase 5 — Protocole diversification (2e business)

**Uniquement si** l'entrepreneur pose le sujet **et** :

- Le business #1 tourne **sans** que le fondateur y passe **> ~50%** de son temps (délégation / systèmes réels, pas théoriques).

**Étapes type :**

1. Recruter / installer le **remplaçant** du fondateur sur le coeur opérationnel.
2. **Transition** ~3 mois avec métriques claires.
3. **Récupérer du temps** — mesurer avant de lancer #2.
4. Lancer #2 avec la même logique de validation que `bos-find` — pas une copie du business #1 par ego.

Si prérequis non remplis : **refuser poliment** le parallèle et nommer le vrai goulot (souvent digestion / recrutement).

### Phase 6 — Sauvegarder et exécuter

1. **Créer** `workspace/BOS/Output/Scaling_Plan_[date].md` avec : analyse des leviers, palier, roadmap 90j, quick wins, risques & budget, et (si applicable) note sur 2e business.
2. **Mettre à jour** `workspace/BOS/Core/Diagnosis.md` — goulot Chase + levier prioritaire.
3. **Mettre à jour** `workspace/BOS/Core/Actions.md` — semaine en cours en **bold**.
4. **Mettre à jour** `workspace/BOS/Core/Goal.md` si les cibles revenue / horizon changent.
5. **Append** `workspace/BOS/Core/Journal.md` — décision et date du plan.
6. **Proposer l'action la plus petite** pour aujourd'hui.

## Garde-fous

- **Ne JAMAIS ouvrir plusieurs canaux avant d'en maîtriser un** (sauf si le canal actuel est mort — alors pivot explicite, pas dispersion).
- **Ne JAMAIS ignorer la trésorerie** — la croissance consomme du cash avant d'en rendre ; le dire dans le plan.
- **Ne JAMAIS ajouter de complexité prématurée** (2e produit, 2e marché) sans fondations.
- **Ne JAMAIS confondre hausse de CA et scaling** (levier par heure fondateur / systèmes).
- **Ne JAMAIS lancer un 2e business sans avoir libéré du temps mesurable sur le 1er.**
- **Ne JAMAIS pousser l'acquisition si satisfaction produit < 4/5** — renvoyer vers `bos-digestion`.
